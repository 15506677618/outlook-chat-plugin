## 🚀 部署方案：源码部署

### 为什么选择源码部署？

**优点**：
- ✅ 灵活，可以随时调整代码
- ✅ 可以在服务器上重新构建
- ✅ 便于调试和更新
- ✅ 可以使用 Git 进行版本管理
- ✅ 适合生产环境和频繁更新

**部署流程**：
```
本地准备 → 上传脚本 → SSH 登录 → 执行脚本 → 自动部署 → 验证测试
```

## 📋 部署步骤概览

### 第一步：准备工作（本地）

1. **确认 Git 仓库**
   ```bash
   cd d:\UVProjects\outlook-chat-plugin
   git push  # 确保代码已推送到远程仓库
   ```

2. **准备 API Key**
   - 访问：https://cloud.siliconflow.cn/
   - 获取 SiliconFlow API Key

3. **获取服务器信息**
   - 阿里云 ECS 公网 IP
   - SSH 登录凭证（密码或密钥）

### 第二步：上传部署脚本

```powershell
# 在本地 PowerShell 执行
scp deploy.sh root@你的服务器 IP:/root/
```

### 第三步：执行部署（服务器上）

```bash
# SSH 登录
ssh root@你的服务器 IP

# 执行部署脚本
cd /root
chmod +x deploy.sh
sudo ./deploy.sh
```

按提示输入：
- Git 仓库地址
- SiliconFlow API Key

### 第四步：安装 SSL 证书

```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d koudai.xin -d www.koudai.xin
```

### 第五步：验证部署

访问：https://koudai.xin/chat.html

## 🎯 关键配置说明

### 1. 环境变量（.env）

```env
SILICONFLOW_API_KEY=sk-your-api-key-here
PORT=3000
```

**重要性**：包含敏感的 API Key，不会上传到 Git

### 2. PM2 配置（ecosystem.config.js）

```javascript
{
  name: 'outlook-chat-backend',
  script: 'src/backend/server.js',
  instances: 1,
  max_memory_restart: '500M',
  autorestart: true
}
```

**作用**：管理 Node.js 进程，自动重启，日志记录

### 3. Nginx 配置

位置：`/etc/nginx/sites-available/koudai.xin`

关键配置：
- 前端：`root /var/www/outlook-chat-plugin/dist`
- 后端代理：`proxy_pass http://localhost:3000`
- SSL 证书：自动配置

### 4. 端口分配

- **5173** - Vite 开发服务器（本地开发）
- **3000** - Express 后端服务器
- **80** - HTTP（生产环境）
- **443** - HTTPS（生产环境）
